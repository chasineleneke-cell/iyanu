RentNG DEPLOYMENT CHECKLIST
============================

BEFORE YOU START
[ ] Do you have a GitHub account? (github.com)
[ ] Do you have a Vercel account? (vercel.com)
[ ] Do you have a Railway account? (railway.app)
[ ] Do you have a Neon account? (console.neon.tech)

If not, create these accounts first (2 minutes each).

============================
STEP 1: PREPARE YOUR CODE (2 minutes)
============================

[ ] Push code to GitHub
    - Open source control in VS Code
    - Add all changes
    - Commit message: "Production deployment"
    - Push to main
    
    OR copy your project to GitHub via web interface

After this step:
- Your code is on GitHub
- Vercel and Railway can access it

============================
STEP 2: DATABASE SETUP - NEON (5 minutes)
============================

[ ] Go to https://console.neon.tech
[ ] Create new project
    - Name: rentnng-prod
    - Version: PostgreSQL 15
    - Click Create
[ ] Wait for project to initialize
[ ] Go to Connection strings
[ ] Select Prisma format
[ ] Copy connection string
[ ] PASTE CONNECTION STRING HERE:
    _________________________________________________
    _________________________________________________

Save this string - you'll need it in next step!

After this step:
- Database is created
- Connection string is ready

============================
STEP 3: DEPLOY BACKEND - RAILWAY (12 minutes)
============================

[ ] Go to https://railway.app
[ ] Click "New Project"
[ ] Select "Deploy from GitHub repo"
[ ] Authorize Railway to access GitHub
[ ] Select your RentNG repository
[ ] Click Deploy
[ ] WAIT FOR BUILD TO COMPLETE (2-3 minutes)
    Watch the logs until it says "Deployment successful"

[ ] Go to Variables tab
[ ] Add variable: DATABASE_URL
    Paste your Neon connection string from Step 2

[ ] Add variable: JWT_ACCESS_SECRET
    Value: MySecretKey123456789

[ ] Add variable: JWT_REFRESH_SECRET
    Value: AnotherSecretKey987654321

[ ] Add variable: NODE_ENV
    Value: production

[ ] Go to Deployments
[ ] Click active deployment
[ ] Copy the URL (format: https://xxx.railway.app)
[ ] PASTE RAILWAY BACKEND URL HERE:
    _________________________________________________

[ ] Test backend health:
    Visit: (your-url)/health in browser
    Should see: {"success": true, ...}

After this step:
- Backend is running on Railway
- Database is connected
- API is accessible

============================
STEP 4: DEPLOY FRONTEND - VERCEL (12 minutes)
============================

[ ] Go to https://vercel.com
[ ] Click "Add New" > "Project"
[ ] Find your RentNG repository
[ ] Click "Import"
[ ] Leave settings default (Next.js auto-detected)
[ ] Click "Deploy"
[ ] WAIT FOR BUILD TO COMPLETE (2-3 minutes)

[ ] While deploying, go to Settings
[ ] Environment Variables section
[ ] Add new variable:
    Name: NEXT_PUBLIC_API_URL
    Value: (your-railway-url)/api

    Example: https://rentnng-backend.railway.app/api

[ ] Save
[ ] Trigger redeploy or wait for auto-deploy
[ ] After deployment, copy the Vercel URL
[ ] PASTE VERCEL FRONTEND URL HERE:
    _________________________________________________

[ ] Visit your Vercel URL in browser
[ ] Should see RentNG homepage
[ ] Check browser console for errors (should be none)

After this step:
- Frontend is running on Vercel
- Connected to backend
- All pages accessible

============================
STEP 5: CONNECT FRONTEND & BACKEND (2 minutes)
============================

[ ] Go back to Railway
[ ] Go to Variables
[ ] Add new variable:
    Name: FRONTEND_URL
    Value: (your-vercel-url)

    Example: https://rentnng-prod.vercel.app

[ ] Save
[ ] Railway auto-redeploys (1-2 minutes)
[ ] Check Deployments to confirm

After this step:
- CORS is configured
- Frontend and backend can communicate
- Ready for testing

============================
EMERGENCY ROLLBACK
============================

If something goes wrong:

Railway rollback:
1. Go to Deployments
2. Find previous working version
3. Click menu (...)
4. Click Rollback

Vercel rollback:
1. Go to Deployments
2. Find previous working version
3. Click menu (...)
4. Click Promote to Production

Database issue:
- Neon keeps backups automatically
- Can restore from Neon console
